# Frontend Build

Frontend assets are compiled using **`@wordpress/scripts`** (wp-scripts), which wraps Webpack 5 with a WordPress-optimised default configuration. The setup is extended in `config/webpack/webpack.config.js`.

---

## Entry Points

| Entry | Source | Output | Purpose |
|---|---|---|---|
| `app` | `src/app.js` | `theme/dist/app.js` + `app.css` | Theme frontend assets |
| `editor` | `src/editor.js` | `theme/dist/editor.js` + `editor.css` | Gutenberg editor assets |

Both entries import their corresponding SCSS files, which are extracted into separate CSS files by the MiniCSSExtractPlugin (included in wp-scripts defaults).

---

## Output Location

All compiled assets are written to `app/web/themes/theme/dist/`.

| File | Purpose |
|---|---|
| `dist/app.js` | Compiled theme JavaScript |
| `dist/app.css` | Compiled theme CSS |
| `dist/editor.js` | Compiled editor JavaScript |
| `dist/editor.css` | Compiled editor CSS |
| `dist/app.asset.php` | Version hash + dependency list for `app.*` (used by PHP) |
| `dist/editor.asset.php` | Version hash + dependency list for `editor.*` (used by PHP) |

The `.asset.php` files are generated by `@wordpress/dependency-extraction-webpack-plugin` and are used in `App.php` and `Gutenberg.php` to enqueue assets with automatic cache-busting.

---

## npm Scripts

Defined in `package.json`:

```bash
# Development (watch mode, source maps, fast rebuilds)
pnpm start

# Production build (minified, optimised, no source maps)
pnpm build
```

Both commands use the custom webpack config file via `--config-file=config/webpack/webpack.config.js`.

**Package manager:** `pnpm@10.14.0`

---

## Webpack Configuration

`config/webpack/webpack.config.js` extends the wp-scripts defaults:

```js
const THEME_DIR = path.resolve(process.cwd(), 'app/web/themes/theme');

module.exports = {
    ...defaults,
    entry: {
        app:    'src/app.js',
        editor: 'src/editor.js',
    },
    output: {
        ...defaults.output,
        path: path.resolve(THEME_DIR, 'dist'),
    },
    // Additional rules: image assets (asset/resource), Babel/React JSX
};
```

Additional module rules:
- **Images** (`png|svg|jpg|jpeg|gif`) — Handled as `asset/resource` (copied to `dist/`)
- **JS/JSX** — Processed via `babel-loader` with `@babel/preset-react`

## SCSS Architecture

Source styles live in `src/scss/` and are imported by `app.scss` and `editor.scss`.

```
src/scss/
├── helpers/
│   ├── _variables.scss       # Design tokens (colours, spacing, breakpoints)
│   ├── _functions.scss       # SCSS functions
│   ├── _mixins.scss          # Reusable mixins
│   ├── _normalize.scss       # CSS reset/normalise
│   ├── _media-queries.scss   # Breakpoint mixins
│   ├── _container.scss       # Max-width container utility
│   └── _fonts.scss           # @font-face declarations
├── layout/
│   ├── _base.scss            # Global element styles (body, links, headings)
│   ├── _typography.scss      # Typography scale
│   └── _editor-base.scss     # Editor-specific layout overrides
└── components/
    └── _button.scss          # Button component
```

### `app.scss`

Imports helpers, layout, and component partials for the frontend.

### `editor.scss`

Scoped under `.wp-block-post-content` to mirror frontend styles inside the Gutenberg editor. Imports: normalize, functions, media-queries, mixins, container, editor-base, typography, button.

---

## WordPress Block Styles

`Gutenberg.php` declares `add_theme_support('wp-block-styles')`, which loads WordPress's default block CSS. This ensures core blocks (buttons, images, separators, etc.) look correct without requiring custom styles for each.

The editor stylesheet (`dist/editor.css`) is also registered via `add_editor_style('dist/editor.css')`, so the Gutenberg editor reflects the frontend appearance as closely as possible.

---

## Adding New JS Modules

Add imports to `src/app.js`:

```js
import './modules/slider';
import './modules/accordion';
```

Or create separate entry points in `webpack.config.js` for code-split bundles (e.g. for a page-specific heavy library).

## Design Tokens → CSS Custom Properties

`theme.json` generates CSS custom properties that are available both in SCSS (via `var()`) and in the Gutenberg editor:

```css
/* Generated by theme.json */
--wp--preset--color--primary:     #005f6b;
--wp--preset--font-size--large:   clamp(1.25rem, 2vw, 1.75rem);
--wp--preset--spacing--md:        1.5rem;
--wp--preset--font-family--sans:  'Sansation', sans-serif;
```

Reference these in SCSS to keep frontend and editor styles in sync:

```scss
.my-element {
    color: var(--wp--preset--color--primary);
    padding: var(--wp--preset--spacing--lg);
}
```
